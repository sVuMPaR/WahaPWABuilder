## Автотесты

### Уже есть

```bash
npm run data:test      # парсеры Wahapedia CSV и MFM (скрипты, без фреймворка)
npm run data:validate  # целостность data pack
npm run build          # TypeScript + сборка Vite
```

### Unit-тесты (аналог JUnit)

В Java — **JUnit**. В этом проекте — **[Vitest](https://vitest.dev/)** (тот же синтаксис, что Jest/JUnit: `describe`, `it`, `expect`).

```bash
npm test           # один прогон
npm run test:watch # при разработке
```

Покрыты **чистые функции** без DOM:

- `src/roster/points.ts` — MFM tiers, пересчёт очков
- `src/roster/validation.ts` — Epic Hero, Battleline, Character, лимиты
- `src/roster/leaders.ts` — bodyguard index, attach targets

Это как unit-тесты сервисного слоя в Java, без Selenium.

### E2E (браузер) — опционально позже

Для кликов по UI как в ручном чеклисте — **Playwright** или **Vitest + happy-dom/jsdom** (ограниченно).

Рекомендация: сначала Vitest на логике (быстро, стабильно), E2E — 3–5 критичных сценариев.

### CI

Добавить в GitHub Actions шаг `npm test` перед deploy — тесты гоняются на каждый push в `main`.
